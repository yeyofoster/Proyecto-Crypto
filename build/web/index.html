How AES encryption works?


AES comprises three block ciphers: AES-128, AES-192 and AES-256. Each cipher encrypts and decrypts data in blocks of 128 bits using cryptographic keys of 128-, 192- and 256-bits, respectively. Symmetric (also known as secret-key) ciphers use the same key for encrypting and decrypting, so the sender and the receiver must both know -- and use -- the same secret key. The AES encryption algorithm defines a number of transformations that are to be performed on data stored in an array. The first step of the cipher is to put the data into an array; after which the cipher transformations are repeated over a number of encryption rounds.
The number of rounds is determined by the key length, with 10 rounds for 128-bit keys, 12 rounds for 192-bit keys and 14 rounds for 256-bit keys.

Why RSA?


RSA emerged as a solution to the problem posed by the practical usage of Diffie-Hellman Key Exchange. It required you to first share your key, also there would be unique keys for every person, so we need to use the discrete log, but the groups of order ≤1024 can be determined. Enter RSA. You now need two keys, a public key (encryption) and a private key (decryption). You share the public key with others, and keep private key with you.The public key encrypts the message, and the private key decrypts the message. Encryption is almost a one-way function, unless you apply the decryption algorithm, or use the private key, to decrypt it. This is known as a trapdoor mechanism.
